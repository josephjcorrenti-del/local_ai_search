# local_ai_search

`local_ai_search` is the integration layer between `local_ai` and `local_search`.

The goal is to provide a unified AI and search experience while preserving clear ownership boundaries between projects.

## Current Status

Foundation is complete.

Implemented:

* runtime paths
* configuration
* status command
* doctor command
* structured logging
* ELK integration
* evidence loading
* evidence validation
* evidence formatting
* inspect-evidence command
* inspect-evidence JSON output
* ecosystem status checks
* ecosystem doctor checks

## Architecture

Responsibilities are intentionally separated.

### local_ai

Owns:

```text
- model interaction
- prompt construction
- shell UX
- profiles
- workspaces
```

### local_search

Owns:

```text
- indexing
- retrieval
- search providers
- artifacts
- evidence generation
```

### local_ai_search

Owns:

```text
- integration
- orchestration
- validation
- evidence consumption
```

## Evidence Boundary

The first integration boundary is a subprocess and JSON contract boundary.

`local_ai_search` consumes evidence generated by `local_search`.

Example:

```bash
local-search evidence artifact.json
```

Evidence is validated before use.

Validation includes:

```text
- retrieval_version
- required fields
- bounded evidence size
- result structure
```

Evidence is treated as source data, not instructions.

## Installation

Development setup:

```bash
cd ~/local_ai_search
source .venv/bin/activate

pip install -e ~/local_search
pip install -e ~/ai/local_ai
```

Verify:

```bash
which local-ai-search
which local-search
which local-ai
```

## Commands

### Status

Check the full ecosystem:

```bash
local-ai-search status
```

Checks:

```text
local_ai_search
local_search
local_ai
```

Check only local_ai_search:

```bash
local-ai-search status --self
```

Checks:

```text
local_ai_search
```

```bash
local-ai-search doctor --self
```

## Design Principles

* no hidden retrieval
* no hidden prompt injection
* bounded evidence only
* logs are the source of truth
* explicit integration boundaries
* subprocess contracts before direct imports
* validation before orchestration
